Introduction

Base URL: http://gamepawbuddy.local

This guide covers the essentials for our API.

See code examples on the right. Switch languages using the top tabs or the nav menu on mobile.

Authenticating requests

To authenticate requests, include an Authorization header with the value "Bearer {YOUR_AUTH_KEY}".

All authenticated endpoints are marked with a requires authentication badge in the documentation below.

You can retrieve your token by visiting your dashboard and clicking Generate API token.

Authenticating requests

This endpoint allows users to authenticate and retrieve an API token. You must provide the user's email and password in the request body.

Authenticate the user and issue an API token.

Example request:
curl --request POST \
    "http://gamepawbuddy.local/api/v1/login/user" \
    --header "Content-Type: application/json" \
    --header "Accept: application/json" \
    --data "{
    \"email\": \"bruno@example.com\",
    \"password\": \"00iG9]\\\"4d,2c!Al\"
}"

const url = new URL(
    "http://gamepawbuddy.local/api/v1/login/user"
);

const headers = {
    "Content-Type": "application/json",
    "Accept": "application/json",
};

let body = {
    "email": "bruno@example.com",
    "password": "00iG9]\"4d,2c!Al"
};

fetch(url, {
    method: "POST",
    headers,
    body: JSON.stringify(body),
}).then(response => response.json());
$client = new \GuzzleHttp\Client();
$response = $client->post(
    'http://gamepawbuddy.local/api/v1/login/user',
    [
        'headers' => [
            'Content-Type' => 'application/json',
            'Accept' => 'application/json',
        ],
        'json' => [
            'email' => 'bruno@example.com',
            'password' => '00iG9]"4d,2c!Al',
        ],
    ]
);
$body = $response->getBody();
print_r(json_decode((string) $body));
